&lt;li&gt;&lt;a href="#add"&gt;&lt;b&gt;add&lt;/b&gt;&lt;/a&gt;: Creates a new LDAP entry&lt;/li&gt;

&lt;p/&gt;
In order to be able to use any of the operations listed before, you must define a &lt;a href="#config"&gt;&lt;b&gt;config&lt;/b&gt;&lt;/a&gt; element with the LDAP connection parameters:

&lt;table&gt;
&lt;tr&gt;
&lt;td&gt;&lt;b&gt;URL&lt;/b&gt;&lt;/td&gt;
&lt;td&gt;
The connection URL to the LDAP server. LDAP connection URLs have the following syntax: &lt;code&gt;ldap[s]://hostname:port/base_dn&lt;/code&gt;
&lt;p/&gt;
&lt;ul&gt;
&lt;li&gt;&lt;b&gt;hostname&lt;/b&gt;: Name (or IP address in dotted format) of the LDAP server. For example, ldap.example.com or 192.202.185.90.&lt;/li&gt;
&lt;li&gt;&lt;b&gt;port&lt;/b&gt;: Port number of the LDAP server (for example, 696). If no port is specified, the standard LDAP port (389) or LDAPS port (636) is used.&lt;/li&gt;
&lt;li&gt;&lt;b&gt;base_dn&lt;/b&gt;: distinguished name (DN) of an entry in the directory. This DN identifies the entry that is the starting point of the search. If no base DN is specified, the search starts at the root of the directory tree.&lt;/li&gt;
&lt;/ul&gt;

Some examples are:
&lt;ul&gt;
&lt;li&gt;ldap://localhost:389/&lt;/i&gt;
&lt;li&gt;ldap://localhost:389/dc=mulesoft,dc=org&lt;/i&gt;
&lt;li&gt;ldaps://localhost:636/dc=mulesoft,dc=org&lt;/i&gt;
&lt;li&gt;ldaps://ldap.mulesoft.org/&lt;/i&gt;
&lt;/ul&gt;
&lt;/td&gt;
&lt;/tr&gt;
&lt;tr&gt;
&lt;td&gt;&lt;b&gt;Type&lt;/b&gt;&lt;/td&gt;
&lt;td&gt;
The implementation of the connection to be used. Right now the only available implementation is JNDI, though any other
implementation can be used (For example using Novell libraries). If you want to create your own implementation you should
extend the class 
&lt;ul&gt;
&lt;li&gt;&lt;b&gt;JNDI&lt;/b&gt;: Implementation that uses the JNDI interfaces provided in the standard JRE.&lt;/i&gt;
&lt;/ul&gt;
&lt;/td&gt;
&lt;/tr&gt;
&lt;tr&gt;
&lt;td&gt;&lt;b&gt;Initial Pool Size&lt;/b&gt;&lt;/td&gt;
&lt;td&gt;
The string representation of an integer that represents the number of connections per connection identity to create when initially
creating a connection for the identity. To disable pooling, just set this value to 0 (zero).
&lt;/td&gt;
&lt;/tr&gt;
&lt;tr&gt;
&lt;td&gt;&lt;b&gt;Max Pool Size&lt;/b&gt;&lt;/td&gt;
&lt;td&gt;
The string representation of an integer that represents the maximum number of connections per connection identity that can be maintained
concurrently.
&lt;/td&gt;
&lt;/tr&gt;
&lt;tr&gt;
&lt;td&gt;&lt;b&gt;Pool Timeout&lt;/b&gt;&lt;/td&gt;
&lt;td&gt;
The string representation of an integer that represents the number of milliseconds that an idle connection may remain in the pool without
being closed and removed from the pool.
&lt;/td&gt;
&lt;/tr&gt;
&lt;tr&gt;
&lt;td&gt;&lt;b&gt;Referral&lt;/b&gt;&lt;/td&gt;
&lt;td&gt;
Constant that holds the name of the environment property for specifying how referrals encountered by the service provider are to be processed.
The value of the property is one of the following strings:
&lt;ul&gt;
&lt;li&gt;&lt;b&gt;follow&lt;/b&gt;: Follow referrals automatically&lt;/li&gt;
&lt;li&gt;&lt;b&gt;ignore&lt;/b&gt;: Ignore referrals&lt;/li&gt;
&lt;li&gt;&lt;b&gt;throw&lt;/b&gt;: Throw ReferralException when a referral is encountered.&lt;/li&gt;
&lt;/ul&gt;
&lt;/td&gt;
&lt;/tr&gt;
&lt;tr&gt;
&lt;td&gt;&lt;b&gt;Extended Configuration&lt;/b&gt;&lt;/td&gt;
&lt;td&gt;
This is a  instance holding extended configuration attributes that will be used in the Context environment.
Values configured here have less precedence than the other values that are allowed
in the module configuration.
Some examples of extended properties (key: value) are:
&lt;ul&gt;
&lt;li&gt;&lt;b&gt;java.naming.language&lt;/b&gt;: Constant that holds the name of the environment
property for specifying the preferred language to use with the service.
The value of the property is a colon-separated list of language tags as
defined in RFC 1766.&lt;/li&gt;
&lt;li&gt;&lt;b&gt;java.naming.security.authentication&lt;/b&gt;: Constant that holds the name of the environment
property for specifying the security level to use. Its value is one of the following
strings: "none", "simple", "strong".&lt;/li&gt;
&lt;li&gt;&lt;b&gt;java.naming.security.protocol&lt;/b&gt;: Constant that holds the name of the environment property for specifying
the security protocol to use. Its value is a string determined by the service provider (e.g. "ssl").&lt;/li&gt;
&lt;li&gt;&lt;b&gt;com.sun.jndi.ldap.connect.pool.authentication&lt;/b&gt;: A list of space-separated authentication types of connections that may be
pooled. Valid types are "none", "simple", and "DIGEST-MD5".&lt;/li&gt;
&lt;li&gt;&lt;b&gt;com.sun.jndi.ldap.connect.pool.debug&lt;/b&gt;: A string that indicates the level of debug output to produce. Valid values are "fine"
(trace connection creation and removal) and "all" (all debugging information).&lt;/li&gt;
&lt;li&gt;&lt;b&gt;com.sun.jndi.ldap.connect.pool.prefsize&lt;/b&gt;: The string representation of an integer that represents the preferred number of
connections per connection identity that should be maintained concurrently.&lt;/li&gt;
&lt;li&gt;&lt;b&gt;com.sun.jndi.ldap.connect.pool.protocol&lt;/b&gt;: A list of space-separated protocol types of connections that may be pooled. Valid types are "plain" and "ssl".&lt;/li&gt;
&lt;/ul&gt;
&lt;/td&gt;
